## 1. 要解決的問題與目標使用者

**目標使用者**：自主操作的加密貨幣散戶投資人——會自己看盤、自己下決定，但資訊來源零散、容易在進場前忽略風險。

**要解決的問題**：散戶在「決定要不要進場某個加密貨幣」之前，常常：
- 只看到單一面向（例如只看價格或只聽消息），忽略其他風險訊號。
- 不知道近期有哪些利空新聞或市場情緒變化。
- 沒有把自己的筆記/歷史判斷跟即時資訊放在一起看。

**產品要提供的價值**：在使用者進場前，用 AI 把**即時行情、新聞情緒、個人知識庫**整合成一份「他可能忽略的風險與盲點 ＋ 正反觀點」，並附上資料來源與時間，協助他**自行做出更周全的判斷**。

---

## 2. 產品定位

加密貨幣**綜合研究助手**：整合行情、新聞、個人知識的查詢與問答工具，以「**進場前的風險/盲點提醒**」為差異化亮點。

**核心立場（產品行為，非技術）**：
- 中立第二意見：**不報明牌、不給「該買/該賣」的明確指令、不保證獲利**。
- 給材料不給結論：列出正反觀點與風險，由使用者自行判斷。
- 每個判斷都要能追溯到資料來源與時間。

---

## 3. 北極星指標（North Star Metric）

**每週完成「深度幣種研究 session」的數量。**

一次「深度幣種研究 session」指使用者**完整走過行為路徑**：市場總覽 → 選定幣種 → 進個幣頁 → 向 AI 提問 → **至少追問一次（回答輪數 ≥ 2）**。所有設計決策都應指向把這個數字做大。

```
深度研究 sessions/week = 週訪客數 × 進入個幣頁率 × AI 問答觸發率 × AI 追問率
```

> P1 **不**內建這個指標的量測（無 DB、無分析工具）；指標先作為設計北極星。實際儀器化在 **P4（GA4）** 或 **P2（DB 對話歷史）** 才建立。詳見 §6。

---

## 4. 核心功能清單（P1 = 第一個可 demo 的「產品」版本）

> P1 = 一個能跑起來、能 demo、能展示 AI 能力的完整產品。含以下四項功能。
> **P1 刻意不含**：技術面指標/K 線圖、風險彙整卡、資料庫、👍/👎 回饋紀錄、觀測分析——全數後移（見 §6）。

### F1. 市場總覽
使用者打開產品，先掌握整體加密市場狀態，並能由此進入任一幣的分析。

### F2. 個幣分析頁（精簡版）
使用者選定一個幣後，在同一頁看到該幣的**行情與近期新聞情緒**，作為進場前的快速體檢，並有明確入口向 AI 提問。

### F3. AI 問答（情境感知）
使用者針對某個幣提問（例如「我現在該進場嗎？」），AI 整合**即時行情、新聞情緒、個人知識庫**三面向，回覆**風險定調＋正反觀點**，每點附來源，結尾附免責，並以串流方式呈現「正在做什麼」。

### F4. 個人知識庫（RAG）
AI 回答時可檢索使用者自有的文字/筆記資料（透過 OpenAI File Search，不自建向量庫），把相關內容納入回答並明確標示「來自你的知識庫」，與公開資料來源區隔。

## 5. 每個功能的可驗證成功條件

以下每句都是「可被測試」的驗收條件。

### F1. 市場總覽
- [ ] 當市場資料可取得時，首頁顯示這幾項：市場情緒指數（恐懼貪婪）、總市值、24h 總成交量、BTC 市佔率、漲幅榜、跌幅榜、市值排行清單。
- [ ] 每個漲跌數值以「綠漲紅跌」呈現，且同時有方向圖示（不只靠顏色）。
- [ ] 點擊清單中任一幣，可導向該幣的個幣分析頁。
- [ ] 當市場資料暫時無法取得時，顯示友善提示訊息，**不崩潰、不顯示捏造數字**。

### F2. 個幣分析頁（精簡版）
- [ ] 頁面顯示該幣：名稱、現價、24h 漲跌、資料更新時間。
- [ ] 顯示近期新聞列表（標題、發布時間、可連到原文）。
- [ ] 每個數據區塊標明資料來源與時間。
- [ ] 提供一個明確入口，讓使用者針對「這個幣」直接向 AI 提問。
- [ ] 任一資料區塊取不到資料時，該區塊顯示友善提示，**其他區塊仍正常顯示**。

### F3. AI 問答（情境感知）
- [ ] 對話開場明確說明能力範圍（看風險/新聞/個人知識），並標示「AI 生成、非投資建議」；**不使用「問我任何事」式的空泛開場**。
- [ ] 從個幣頁發問時，AI 知道目前討論的是哪一個幣（情境感知），無需使用者再次指明。
- [ ] AI 整合三面向資料：即時行情、近期新聞、個人知識庫；新聞情緒由 AI 依標題判讀（利多/利空/中性）並標明來源與時間。
- [ ] AI 回答**先給結論**（風險定調），再分「正面觀點 / 風險與盲點」分點呈現。
- [ ] 回答中每個關鍵判斷都附上資料來源；數字僅來自實際資料，**不得自行編造**。
- [ ] 回答結尾固定出現免責聲明（非投資建議、請自行查證）。
- [ ] AI **不輸出**明確的「買進/賣出」指令或保證獲利的字眼。
- [ ] 回答以串流方式逐步顯示，且過程中讓使用者看到 AI 正在做什麼（如「取得行情 → 檢索新聞 → 查知識庫 → 彙整中」步驟）。
- [ ] 提供與當前幣相關的建議追問選項，使用者點擊即可延續對話。

### F4. 個人知識庫（RAG）
- [ ] 當問題與使用者知識庫相關時，AI 回答會引用知識庫內容，並標明「來自你的知識庫」。
- [ ] 知識庫來源與公開資料來源在回答中可區分（不同標示）。
- [ ] 知識庫無相關內容或檢索失敗時，AI 不編造，照常以公開資料作答。

### 全產品通用成功條件
- [ ] 全站漲跌色彩一致（綠漲紅跌），狀態以「圖示＋文字＋顏色」呈現。
- [ ] 所有對外顯示的數據都可追溯到來源與時間。
- [ ] 產品可成功部署到 Railway，並透過公開網址正常開啟與操作（市場總覽 → 個幣頁 → AI 問答）。
- [ ] 外部 API 額度受保護：以記憶體快取（Map + TTL）避免被免費 API 限流（429），限流時回上次成功資料而非崩潰。

## 6. 分期路線圖（P1 結案才談 P2）

> 原則：**P1 是一個能跑起來、能 demo 的完整產品**；P1 沒做完，後面的風險彙整卡、File Search 進階、GA4 全都沒意義。P2 是作品集的差異化關鍵；P3/P4 有時間多就做。

| 階段 | 一句話 | 主要內容 |
|------|--------|---------|
| **P1（能 Demo）** | 最小可展示產品 | 3 個頁面（市場總覽 / 個幣頁 / AI 問答）+ 3 支 AI 工具（即時行情 / 新聞 / 知識庫）+ 記憶體 API 快取 + 串流 UI + 免責 + 錯誤處理。部署 Railway。**不需要 DB。** |
| **P2（有亮點）** | 差異化 | 風險彙整卡（風險定調＋信心標籤＋正反分點＋來源）、個幣頁加技術面（RSI/MACD/均線）+ K 線圖、新增 4 支工具（getOHLCV / calcTechnicalSignals / analyzeSentiment / getOnChainBasics）、**Railway PostgreSQL + Drizzle ORM**（對話歷史 `chat_sessions`/`messages`＋風險歷史 `risk_assessments`/`risk_points`）、👍/👎 回饋紀錄。 |
| **P3（有深度）** | 更深分析 | 細緻歷史快照（`technical_signals`/`sentiment_snapshots`/`onchain_snapshots`）、工具執行追蹤（`tool_executions`）、新聞/K 線/市場快照等各式快取。 |
| **P4（有數據）** | 觀測與優化 | **GA4 埋點**（6 個事件 + 4 個自訂維度）、漏斗/趨勢/熱門幣/留存探索報表（量測 §3 北極星指標）、Upstash Redis 熱資料快取。 |

## 7. 不做什麼（YAGNI / 非目標）

**全產品永遠不做：**
- ❌ 不做使用者帳號、登入、金流、付費牆（數據以匿名 session 記錄即可）。
- ❌ 不做自動下單，或連接交易所帳戶進行交易。
- ❌ 不提供「該買/該賣」的明確投資指令或目標價，不保證獲利。
- ❌ 不做模型微調（fine-tuning）；個人資料一律走 RAG 檢索（OpenAI File Search），不自建向量庫。
- ❌ 不追求企業級健壯度（高可用、嚴格 SLA）；以可 demo、可展示為準。

**P1 不做（後移到指定階段）：**
- ❌ 資料庫（對話歷史、行為紀錄、風險歷史）→ **P2** 起用 Railway PostgreSQL。
- ❌ 技術面指標（RSI/MACD/均線）與 K 線圖 → **P2**。
- ❌ 風險彙整卡（整合多面向成單一結構化卡片）→ **P2**。
- ❌ 👍/👎 回饋紀錄（需 DB）→ **P2**。
- ❌ 鏈上/基本面數據 → **P2** 起。
- ❌ 觀測數據 dashboard 與 GA4 埋點 → **P4**。
- ❌ 影片/圖片等多模態知識庫資料 → 後期。

## 8. 法規與免責（產品行為要求）

- 採保守路線：提供風險資訊與一般性分析，**不推介特定標的買賣、不保證獲利**。
- 全程顯著免責，且採「情境式免責」——放在使用者做判斷的決策點旁，而非到處貼被忽略的小字。
